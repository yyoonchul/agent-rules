# 작업 지침

다음 지침은 `시험_필수암기_정리.md` cram sheet를 만들 때 반드시 적용한다.

## 1. 범위 고정

- 입력은 항상 **폴더 하나**. 해당 폴더 + 1레벨 하위(`HW*`, `hw*`, `예상문제_이미지/` 등 시험 범위가 명확한 경우)까지만 스캔한다.
- 폴더 밖 자료는 사용하지 않는다. 폴더에 없는 개념/문제는 cram sheet에 넣지 않는다.

## 2. 자료 분류 (라벨 붙이기)

| 라벨 | 예시 파일명 패턴 | 역할 |
| --- | --- | --- |
| **digest** | `*_ digest.md`, `CH*_digest*`, `lecture-notes/` | 개념 원천 (정의·공식의 ground truth) |
| **past-exam** | `Quiz*`, `sample_midterm*`, `*_Answer*`, `기출*` | 시험 패턴 역분석의 **1순위 근거** |
| **homework** | `HW*`, `hw*`, `AS*` | 계산 패턴·수치 답 형식의 2순위 근거 |
| **example** | `예제문제*`, `예상문제*`, `연습문제*`, `problem_set*` | 3순위 근거 (past-exam 부족 시 보완) |
| **raw-slide** | `*.pdf`, `*.pptx` (digest가 없는 경우만) | digest 미존재 시 fallback |

- `raw-slide`는 같은 이름의 `_ digest.md`가 있으면 digest를 우선한다.

## 3. 역분석 절차

1. past-exam / homework / example 각 파일에서 **모든 문제**를 열거한다.
2. 각 문제에 태그: `{chapter, type, keywords, evidence-id}`. evidence-id는 `Q1-4`, `Sample P2`, `HW1 P5`, `예제 3-2` 등 사용자가 바로 추적 가능한 레이블을 쓴다.
3. **같은 archetype으로 묶고** 빈도 순으로 정렬한다.
4. 2번 이상 반복된 **오답 유발 포인트**는 "함정"으로 분리한다.
5. 어느 past-exam 또는 sample에서 **유도 전체**를 요구한 항목은 "반드시 유도할 줄 알아야 할 수식"으로 격상한다.

## 4. 섹션별 작성 규칙

### 섹션 1 · 출제 패턴
- 표: `유형 | 설명 | 근거`. 근거 열에는 evidence-id를 나열.
- 관찰 아래에 **추론 3~5개** (관찰된 사실로부터 "∴ 이런 유형이 또 나올 것"을 도출). 추론에는 `⚠️` 없이 쓰되, 근거 없는 추측은 금지.

### 섹션 2 · 챕터별 필수 암기 (문서 본체, 설명 밀도 최우선)

이 섹션은 **"처음 공부하는 사람도 이것만 읽고 시작할 수 있는 자족형 학습 문서"** 여야 한다. 단순 개념 나열·불릿 집합은 실패 출력이다.

챕터별로 아래 3단:

- `#### 큰 그림` — 3~6문장의 **산문**. 이 챕터가 왜 존재하는지, 어떤 도구 세트를 제공하는지, 이전/이후 챕터와 어떻게 이어지는지. 불릿 금지.

- `#### 핵심 개념` — 개념마다 `**(n) 제목**` 블록에 아래 요소를 **모두** 포함. 요소가 빠졌다면 그 개념은 아직 쓰다 만 것이다.
  1. **정의** (1~3 문장, 한국어). 기술용어는 English.
  2. **핵심 수식 / 공식 / 시그니처** — KaTeX 블록. 수식 바로 아래에 **기호 풀이**: 각 기호가 무엇이고, 단위/차원/기본값/제약은 무엇인지. 기호 풀이 없이 수식만 두지 말 것.
  3. **왜 그런가** — "왜 마이너스?", "왜 $2\theta$?", "왜 이 함수가 `int` 반환?" 같이 **스스로 질문을 던지고 답한다**. 짧은 유도 스케치 또는 직관적 설명. 최소 한 문단.
  4. **예시 또는 비유** — 이해를 돕는 짧은 예시(수치 포함 권장) 또는 digest의 비유 재활용.
  5. **헷갈리기 쉬운 포인트** — sign convention, 차원, 단위, 범위, 경계 조건 등. 시험 함정과 연결되는 자리.
  6. **연결 개념** — 이 개념이 어느 English term과 묶이는지, 다음 개념에서 어떻게 쓰이는지 한 줄.
  7. **시험 근거** — 과거에 등장한 evidence-id (`HW1 P5`, `Q2-3`, `Sample P4(b)`, `예제 3-2`). 없으면 "직접 근거 없음, 챕터 필수 배경"으로 표기.

  개념 순서는 digest의 흐름을 따르되, **past-exam에서 실제로 출제된 개념을 앞에 또는 더 길게** 배치한다.

- `#### 암기 포인트` — `- [ ]` 체크박스로 위 본문의 **압축 요약만**. 암기 포인트가 본문을 대체하지 않도록 한다 (본문을 이미 읽은 사람이 시험 직전에 스캔하는 용도).

**분량/밀도 규칙**:
- 개념 하나당 설명 본문은 적어도 digest 동일 항목과 **같은 분량**, 출제 가능성이 높을수록 **더 길게**. 과거 자료 한 줄이었더라도 시험 예상 개념이면 여기서는 문단으로 풀어 쓴다.
- 반대로, 과거 자료에 **전혀 등장하지 않은** 세부 개념은 이 섹션에서 제외 (depth over breadth).
- 본문 판정 기준: "이 섹션만 읽고 시험장 갈 수 있나?" — YES가 아니면 더 풀어 써야 한다.

**문체**:
- 설명은 한국어 산문, 기술 용어는 English 원어 (`stress transformation`, `VC dimension`, `O_APPEND`, `backprop`, `page fault`).
- 수식은 KaTeX. 여러 줄이면 `$$` 블록. 수식을 썼으면 기호 풀이·단위·제약을 반드시 병기.
- "왜?"를 최소 한 문장. 정의만 나열하면 실패.
- past-exam/HW에서 나왔으면 그 문제 번호를 **본문에 인라인으로** 붙인다 (예: "*HW1 P5에 직접 출제*").

### 섹션 3 · 반드시 유도할 줄 알아야 할 수식
- 수학·통계·물리 계열 과목에서만 포함.
- 각 항목: 3~6줄로 유도 과정 축약본 (전체 풀이 아닌 흐름 + 핵심 식).
- sample exam 또는 quiz에서 유도 요구가 실제로 있었던 것만 최상위로.

### 섹션 4 · 함정 포인트 TOP
- 표: `# | 함정 진술 | 올바른 답 | 근거`.
- 근거 없는 함정은 쓰지 않는다. 대신 `⚠️`를 붙이고 "digest에 명시는 약하지만 자주 혼동" 같은 식으로 이유 설명.
- 최소 10개, 가능하면 20개 이상.

### 섹션 5 · 시험 직전 치트시트
- 번호 리스트. 각 항목은 **한 줄**, 공식·플래그·커맨드·명령어 조합 하나.
- 책을 펴 놓고도 바로 긁어 쓸 수 있는 형태 (예: `O_WRONLY|O_CREAT|O_TRUNC`, `$\tau_{\max} = \sqrt{((\sigma_x-\sigma_y)/2)^2 + \tau_{xy}^2}$`).
- 15개 이상.

### 섹션 6 · 시험장 전략 메모 (선택)
- 서술형/계산형/T-F 각각에 대한 답변 구조 팁을 3~5불릿.
- 과목 성격에 안 맞으면 생략.

### 섹션 7 · 시험 직전 체크리스트 (선택)
- 하위 그룹: `반드시 암기` / `설명 연습` / `계산·트레이스` / `코드 빈칸` / `버그 이름` 등 과목에 맞는 것만.
- 각 항목은 `- [ ]`.

## 5. 용어 / 언어

- 전문 용어 English 유지, 설명은 한국어.
- 한국어 신조어 만들지 않는다 (`support vector`를 "지지 벡터"로 바꾸지 말 것).

## 6. 증거 추적

- 모든 "출제 예상 / 고빈도 / 함정" 주장은 폴더 내 실제 근거 파일에 매핑 가능해야 한다.
- 근거 불충분한 추론은 `⚠️`로 표시하고 근거 부족 사유 한 줄을 덧붙인다.

## 7. 불확실성 처리

- past-exam이 하나도 없으면: 출제 패턴 섹션은 "직접 근거 부족, example 기반 추론" 명시 후 보수적으로 작성.
- 추출 실패한 PDF가 있으면: cram sheet 끝에 `## Uncertainty Notes`를 붙여 어떤 자료가 읽히지 않았는지 기재.
- 존재하지 않는 past-exam 문제, 함정, 공식은 절대 만들어내지 않는다.
