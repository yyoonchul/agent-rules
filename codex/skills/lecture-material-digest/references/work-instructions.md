# 작업 지침

다음 지침은 강의자료 다이제스트를 만들 때 반드시 적용한다.

1. 범위 고정
- 입력 파일은 1개 이상일 수 있으며, 여러 개면 순서대로 처리한다.
- 자료 전체를 훑고 누락 없이 주요 개념을 뽑는다.

2. 용어 보존
- 원문에 있는 핵심 용어, 모델명, 알고리즘명, 수식명은 English 그대로 유지한다.
- 불필요한 한글 번역 용어를 새로 만들지 않는다.

3. 설명 언어
- 설명 문장은 한국어로 작성하되, 학습자가 이해할 수 있을 만큼 충분히 자세히 쓴다.
- 지나치게 짧은 요약형 문장만 나열하지 않는다.

4. 개념 설명 깊이
- 각 주요 개념마다 최소한 다음을 포함한다: "무엇인지(정의)", "왜 중요한지(배경/필요성)", "어떻게 동작하는지(절차/원리)", "예시 또는 비유".
- 학습자가 혼동하기 쉬운 포인트와 구분 기준을 함께 적는다.

5. 개념 구조화
- 유사 개념을 묶어 섹션화하고, 기초 → 심화 흐름으로 배치한다.
- 개념 간 연결 관계(선행 개념, 비교 개념, 함께 쓰는 개념)를 명시한다.

6. 불확실성 처리
- 텍스트 추출이 불완전하면 해당 범위를 `불확실`로 명시한다.
- 추정으로 내용을 채우지 않는다.
