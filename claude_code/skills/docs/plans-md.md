# Planning Guide (PLANS.md)

## Planning Types

### Light Plans (Tasks < 1 day)
- No separate document needed
- Record intent in commit messages or PR descriptions

### Complex Plans (Tasks > 2 days)
- Create a detailed plan before starting
- Break down work into small, testable steps
- Use a tracking issue or a dedicated markdown file if needed

## Planning Checklist

- [ ] Is the goal clear?
- [ ] Is the scope (in/out) defined?
- [ ] Are there completion criteria for each step?
- [ ] Have risks been identified?
- [ ] Is a relevant design document linked?
